# Security Policy

## Reporting a Vulnerability

If you discover a security vulnerability in mcplint, please report it responsibly.

**Do NOT open a public GitHub issue for security vulnerabilities.**

Instead, please email: **security@mcplint.dev**

Or use [GitHub's private vulnerability reporting](https://github.com/mcplint/mcplint/security/advisories/new).

### What to include

- Description of the vulnerability
- Steps to reproduce
- Impact assessment (what an attacker could achieve)
- Any suggested fixes (optional)

### Response timeline

- **Acknowledgment** within 48 hours
- **Initial assessment** within 7 days
- **Fix or mitigation** as soon as practical, depending on severity

## Scope

This policy covers the mcplint codebase and its published artifacts (binaries, crate packages). It does not cover third-party dependencies — please report those to their respective maintainers.
